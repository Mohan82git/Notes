### **HTML Roadmap (1 Month)**

**Goal:** Master **HTML** fundamentals and best practices to build well-structured web pages for **WordPress development & PHP projects**.

---

## **Phase 1: HTML Basics (1 Week)**

### **Topics to Cover**

✔ Introduction to HTML (`<html>`, `<head>`, `<body>`)  
✔ Text Formatting (`<h1>-<h6>`, `<p>`, `<strong>`, `<em>`)  
✔ Lists (`<ul>`, `<ol>`, `<li>`)  
✔ Links (`<a>` with `href`, `target="_blank"`)  
✔ Images (`<img>` with `src`, `alt`, `width`, `height`)

### **Hands-on Tasks**

✅ Create a **simple webpage** with a title, headings, and paragraphs  
✅ Add **lists & links** to navigate between pages  
✅ Display **images with alt text** for accessibility

### **Common Mistakes to Avoid**

❌ Not closing tags properly (`</p>`, `</h1>`)  
❌ Using `<br>` multiple times instead of **CSS for spacing**  
❌ Forgetting `alt` attributes for images (affects accessibility & SEO)

---

## **Phase 2: HTML Forms & Tables (1 Week)**

### **Topics to Cover**

✔ Forms (`<form>`, `<input>`, `type`, `placeholder`, `label`)  
✔ Buttons & Form Controls (`<button>`, `<select>`, `<textarea>`, `checkbox`, `radio`)  
✔ Tables (`<table>`, `<tr>`, `<th>`, `<td>`, `colspan`, `rowspan`)

### **Hands-on Tasks**

✅ Create a **contact form** with name, email, and message fields  
✅ Design a **basic login form** with username & password fields  
✅ Build a **simple table** to display product details

### **Common Mistakes to Avoid**

❌ Not using **labels** for form elements (`<label for="email">`)  
❌ Forgetting to group related inputs with `<fieldset>`  
❌ Using `<table>` for layout instead of **CSS Grid/Flexbox**

---

## **Phase 3: HTML5 & Semantic Elements (1 Week)**

### **Topics to Cover**

✔ HTML5 Semantic Tags (`<header>`, `<nav>`, `<section>`, `<article>`, `<footer>`)  
✔ Audio & Video (`<audio>`, `<video>`, `controls`, `autoplay`)  
✔ Forms with HTML5 Validation (`required`, `pattern`, `minlength`, `maxlength`)  
✔ Metadata & SEO (`<meta>`, `viewport`, `title`, `description`)

### **Hands-on Tasks**

✅ Redesign a webpage using **semantic elements** instead of `<div>`  
✅ Embed an **audio & video player** on a webpage  
✅ Implement a **responsive form** with validation

### **Common Mistakes to Avoid**

❌ Using **divs everywhere** instead of semantic elements  
❌ Forgetting **meta tags for SEO & mobile responsiveness**  
❌ Ignoring HTML5 form validation (makes backend validation necessary)

---

## **Phase 4: Accessibility, Best Practices & Project (1 Week)**

### **Topics to Cover**

✔ ARIA Roles & Attributes (`aria-label`, `role="navigation"`)  
✔ Keyboard Navigation & Screen Readers  
✔ HTML Performance Optimization (Lazy Loading, Minification)

### **Final Project**

✅ Build a **responsive multi-page website** with:

- **Semantic HTML structure**
- **Navigation bar & footer**
- **Forms, tables, images, & media**

### **Common Mistakes to Avoid**

❌ Ignoring **accessibility features** (makes sites unusable for disabled users)  
❌ Using `<iframe>` excessively (bad for performance & security)  
❌ Not testing on **multiple devices** (desktop, mobile, tablet)

---

### **Final Checklist for HTML Mastery**

✔ **Basic Structure:** Headings, paragraphs, lists, links, images  
✔ **Forms & Tables:** Input fields, validation, tables for structured data  
✔ **HTML5 Features:** Semantic tags, media, metadata for SEO  
✔ **Accessibility & Performance:** ARIA roles, lazy loading, best practices  
✔ **Complete Project:** A well-structured, responsive website
