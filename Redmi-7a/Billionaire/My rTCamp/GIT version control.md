# **Roadmap for Mastering Git Version Control (2.5 Months)**

**Goal:** Learn **Git version control** for managing projects, collaborating with teams, and following best practices for professional software development.

---

## **Phase 1: Git Basics & Local Workflow (3 Weeks)**

### **Topics to Cover**

✔ What is Git? How It Works (`Working Directory`, `Staging Area`, `Repository`)  
✔ Installing & Configuring Git (`git config`, `git init`)  
✔ Basic Commands (`git add`, `git commit`, `git status`, `git log`)  
✔ Understanding **Branches** (`git branch`, `git checkout`, `git switch`)  
✔ Cloning & Working with Remote Repos (`git clone`, `git push`, `git pull`)

### **Hands-on Tasks**

✅ Initialize a **Git repository** and commit code changes  
✅ Create and **switch between branches**  
✅ Clone a **GitHub repository and push changes**

### **Common Mistakes to Avoid**

❌ Committing **large, unnecessary files** (use `.gitignore`)  
❌ Using **vague commit messages** (`Updated code` instead of `Fixed login bug`)  
❌ Forgetting to **stage files** before committing (`git add .`)

---

## **Phase 2: Collaboration & Advanced Git Commands (4 Weeks)**

### **Topics to Cover**

✔ Working with **Remotes & Forks** (`git remote`, `git fork`, `git fetch`)  
✔ Merging & Resolving Conflicts (`git merge`, `git rebase`)  
✔ Pull Requests & Code Reviews (GitHub, GitLab, Bitbucket)  
✔ Resetting & Reverting (`git reset`, `git revert`, `git stash`)  
✔ Git Tagging (`git tag`, `git describe`) for release versions

### **Hands-on Tasks**

✅ Create a **feature branch and merge it back to the main branch**  
✅ Resolve a **merge conflict using `git merge` or `git rebase`**  
✅ Submit a **pull request (PR) on GitHub and request a review**

### **Common Mistakes to Avoid**

❌ Not **creating a new branch for each feature/fix** (Avoid working directly on `main`)  
❌ Using **`git pull` without checking differences**, causing conflicts  
❌ Force pushing (`git push --force`) without understanding the risk

---

## **Phase 3: Git Best Practices & Automation (4 Weeks)**

### **Topics to Cover**

✔ Writing **Better Commit Messages** (`feat:`, `fix:`, `docs:`)  
✔ Squashing Commits (`git rebase -i`) for a clean history  
✔ Cherry-picking specific commits (`git cherry-pick`)  
✔ Automating Workflows with **Git Hooks** (`pre-commit`, `post-merge`)  
✔ Using **CI/CD with GitHub Actions**

### **Hands-on Tasks**

✅ Squash multiple commits into a **single clean commit**  
✅ Use `git cherry-pick` to **apply a specific commit to another branch**  
✅ Set up **a pre-commit hook to check code formatting**

### **Common Mistakes to Avoid**

❌ Not **reviewing changes before committing** (`git diff`, `git status`)  
❌ Squashing commits **when history clarity is needed**  
❌ Forgetting to **pull latest changes before pushing**

---

## **Final Project: Git-Based Team Collaboration (3 Weeks)**

✅ **Choose one:**  
1️⃣ **Contribute to an open-source project using GitHub pull requests**  
2️⃣ **Work on a team project using feature branches & merge workflows**  
3️⃣ **Set up Git hooks for automated code formatting & commit checks**

---

## **Final Checklist for Git Version Control Mastery**

✔ **Local Workflow:** Commits, branches, merges, and rebases  
✔ **Collaboration:** Pull requests, conflict resolution, and forks  
✔ **Best Practices:** Clear commit messages, clean history, Git hooks  
✔ **Automation:** Pre-commit hooks & GitHub Actions  
✔ **Final Project:** Successfully collaborated on a Git-managed project
