# **Roadmap for Technical Documentation & Git Commits (1.5 Months)**

**Goal:** Learn to write **clear technical documentation** (README, API docs, code comments) and follow **best practices for Git commits** to maintain a professional codebase.

---

## **Phase 1: Writing Technical Documentation (3 Weeks)**

### **Topics to Cover**

✔ Importance of Documentation (Why it Matters)  
✔ README Structure (`Introduction`, `Installation`, `Usage`, `Contributing`, `License`)  
✔ Writing API Documentation (`Endpoints`, `Request/Response`, `Examples`)  
✔ Code Comments (Inline & Block Comments for Clarity)  
✔ Using **Markdown** for Documentation (`.md` files)

### **Hands-on Tasks**

✅ Create a **README for a PHP/WordPress project**  
✅ Write a **simple API documentation** with sample requests/responses  
✅ Add meaningful **code comments in a JavaScript function**

### **Common Mistakes to Avoid**

❌ **Writing too much or too little** (Keep it concise & relevant)  
❌ **Not using proper formatting** (Use headings, lists, and code blocks in Markdown)  
❌ **Forgetting to update documentation** (It should evolve with the code)

---

## **Phase 2: Writing Effective Git Commits (2 Weeks)**

### **Topics to Cover**

✔ Git Commit Best Practices (Short & Descriptive Messages)  
✔ Commit Message Structure (`Subject`, `Body`, `Footer`)  
✔ Using Conventional Commits (`feat:`, `fix:`, `docs:`, `refactor:`)  
✔ Writing Clear Commit Messages (`Fix login issue`, `Add API support for users`)  
✔ Managing Git History (`git rebase`, `git commit --amend`)

### **Hands-on Tasks**

✅ Write **meaningful commits for a small project**  
✅ Use **Conventional Commits format** in a GitHub repository  
✅ Squash **multiple commits into a clean history**

### **Common Mistakes to Avoid**

❌ Writing **vague commit messages** (`Update file`, `Fix stuff`)  
❌ Making **too many small commits** instead of grouping related changes  
❌ Forgetting to **write commit messages in the imperative form** (`Add feature` instead of `Added feature`)

---

## **Final Project: Well-Documented GitHub Repository (2 Weeks)**

✅ **Choose one:**  
1️⃣ Create a **GitHub repo with a professional README & proper commits**  
2️⃣ Write **API documentation for a PHP project**  
3️⃣ Refactor commits in an existing repo for **better history & readability**

---

## **Final Checklist for Documentation & Git Commits**

✔ **README.md:** Clear, structured, easy to follow  
✔ **API Docs:** Example requests, responses, authentication details  
✔ **Code Comments:** Meaningful, not redundant, helps future developers  
✔ **Git Commits:** Well-structured, follows best practices  
✔ **Final Project:** A fully documented, well-committed repository
