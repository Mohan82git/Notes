### **CSS Roadmap (1.5 Months)**

**Goal:** Master **CSS** to design **responsive, beautiful, and accessible** websites for **WordPress & PHP projects**.

---

## **Phase 1: CSS Basics (1 Week)**

### **Topics to Cover**

✔ CSS Syntax & Selectors (`.class`, `#id`, `*`, `div > p`)  
✔ Colors, Backgrounds & Borders (`color`, `background-color`, `border`)  
✔ Text Styling (`font-family`, `font-size`, `line-height`, `text-align`)  
✔ Box Model (`margin`, `padding`, `border`, `width`, `height`)

### **Hands-on Tasks**

✅ Style a **basic webpage** (Headings, paragraphs, buttons)  
✅ Create **different button styles** (Hover effects, rounded buttons)  
✅ Design a **card UI component** using the **box model**

### **Common Mistakes to Avoid**

❌ Forgetting to **reset margins & paddings** (use `* { margin: 0; padding: 0; }`)  
❌ Using **inline styles (`style=""`)** instead of external CSS files  
❌ Overusing **ID selectors (`#id`)** instead of reusable class selectors

---

## **Phase 2: Layouts with CSS (1 Week)**

### **Topics to Cover**

✔ Positioning (`static`, `relative`, `absolute`, `fixed`, `sticky`)  
✔ Display & Visibility (`block`, `inline-block`, `none`)  
✔ Flexbox (`display: flex`, `justify-content`, `align-items`, `gap`)  
✔ CSS Grid (`display: grid`, `grid-template-columns`, `grid-gap`)

### **Hands-on Tasks**

✅ Build a **flexbox-based navbar**  
✅ Design a **3-column card layout** using **CSS Grid**  
✅ Create a **sticky header & footer**

### **Common Mistakes to Avoid**

❌ Using **too many divs** instead of proper layout techniques  
❌ Not using **CSS Grid/Flexbox** for layouts (Relying on `float` is outdated)  
❌ Forgetting **`overflow: hidden`** when using `position: absolute`

---

## **Phase 3: Responsive Design (1 Week)**

### **Topics to Cover**

✔ Media Queries (`@media screen and (max-width: 768px)`)  
✔ Viewport Units (`vh`, `vw`, `em`, `rem`, `%`)  
✔ Flexbox for Responsive Layouts  
✔ CSS Grid for Mobile-Friendly Websites

### **Hands-on Tasks**

✅ Make a **mobile-responsive navbar**  
✅ Create a **grid-based gallery** that adapts to screen size  
✅ Convert a **multi-column layout into a single-column layout** on mobile

### **Common Mistakes to Avoid**

❌ Ignoring **mobile-first design** (Start styling for mobile first, then larger screens)  
❌ Using **fixed widths (`width: 1000px`)** instead of percentages or `max-width`  
❌ Not testing on **multiple devices**

---

## **Phase 4: Advanced CSS (2 Weeks)**

### **Topics to Cover**

✔ Animations & Transitions (`transition`, `@keyframes`, `animation`)  
✔ CSS Variables (`--primary-color`, `var(--primary-color)`)  
✔ CSS Pseudo-Elements & Pseudo-Classes (`::before`, `::after`, `:hover`, `:nth-child`)  
✔ Dark Mode & Theming (`prefers-color-scheme: dark`)

### **Hands-on Tasks**

✅ Add a **hover effect** to buttons & images  
✅ Create a **loading spinner animation** using only CSS  
✅ Implement a **dark mode toggle** using CSS variables

### **Common Mistakes to Avoid**

❌ Overusing animations (**Too many animations slow down websites**)  
❌ Not using **CSS variables** for theme consistency  
❌ Using **JavaScript unnecessarily** for things that CSS can handle

---

## **Final Phase: Best Practices & Full Project (1.5 Weeks)**

### **Topics to Cover**

✔ CSS Performance Optimization (`minify CSS`, `remove unused styles`)  
✔ CSS Naming Conventions (`BEM methodology`)  
✔ Accessibility (`contrast ratios`, `focus states`)  
✔ Preprocessors (SASS Basics)

### **Final Project**

✅ Build a **multi-page, fully responsive website** that includes:

- A **flexbox navbar**
- A **CSS Grid layout**
- Smooth **animations & transitions**
- A **dark mode toggle**

### **Common Mistakes to Avoid**

❌ Not testing with **contrast checkers** for accessibility  
❌ Ignoring CSS best practices (Organize code, use comments, follow structure)  
❌ Writing **long, unstructured CSS files** (Use modular CSS & reusable classes)

---

### **Final Checklist for CSS Mastery**

✔ **Styled Elements:** Colors, backgrounds, borders, text, buttons  
✔ **Layouts:** Flexbox & Grid for modern responsive design  
✔ **Animations & Effects:** Transitions, hover effects, dark mode  
✔ **Performance & Best Practices:** Minified CSS, accessibility, BEM naming  
✔ **Complete Project:** A polished, responsive, accessible website
